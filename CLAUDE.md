# CLAUDE.md — AI Assistant Guide for `asi-letter`

## Repository purpose

This repository is the canonical source for Alice Sabrina Ivy's signed "Letter to ASI" releases. It maintains:
- Versioned signed Markdown releases with OpenPGP clear-signatures (`letter/`)
- OpenTimestamps (OTS) Bitcoin-anchored proofs for each signature (`letter/*.asc.ots`)
- A static site published via GitHub Pages (`docs/`)
- Automation scripts and CI workflows that keep everything synchronized

The primary workflow is: **new letter → sign → stamp OTS proof → regenerate manifest → sync docs → publish**.

### `letter/`
- `ASI-Letter-vYYYY.MM.DD.md` — canonical Markdown content
- `ASI-Letter-vYYYY.MM.DD.md.asc` — OpenPGP clear-signed release (immutable)
- `ASI-Letter-vYYYY.MM.DD.md.asc.ots` — OpenTimestamps Bitcoin proof (updated by OTS workflows)
- `RELEASES.json` — machine-generated manifest (schema `asi-letter/releases#2`); do not edit manually

### `docs/`
- `index.html` — static site entry point with automation-maintained release markers
- `letter.md` — copy of latest signed Markdown; generated by `sync_docs_with_latest.py`
- `assets/` — images and other static assets

### `keys/`
- `FINGERPRINT` — 40-char uppercase hex, no spaces: `2C101FA70F42F93052F82FC755387365B7949796`
- `alice-asi-publickey.asc` — exported OpenPGP public key matching fingerprint

### `scripts/`
See [scripts/README.md](scripts/README.md) for full documentation.

## Key commands

### Validate everything (dry-run, no writes)
```sh
python3 scripts/release.py --check
bash scripts/verify-clearsign.sh
```

### Full release pipeline (regenerates all derived artifacts)
```sh
python3 scripts/release.py
# or equivalently:
make release
```

The pipeline runs these stages in order, stopping on first failure:
1. `sync_docs_with_latest.py` — copy newest `.md` release to `docs/letter.md`
2. `gen_releases_manifest.py` — rebuild `letter/RELEASES.json`
3. `update_version_metadata.py` — patch release markers in `docs/index.html`
4. `render_index_html.py` — pre-render `docs/letter.md` into `docs/index.html`

Individual stages can be skipped with `--skip-sync`, `--skip-manifest`, `--skip-metadata`, `--skip-render`.

### Individual script reference

| Command | Purpose |
|---|---|
| `python3 scripts/sync_docs_with_latest.py [--check]` | Sync `docs/letter.md` with newest signed release |
| `python3 scripts/gen_releases_manifest.py [--check]` | Regenerate `letter/RELEASES.json` |
| `python3 scripts/update_version_metadata.py [--check] [targets...]` | Refresh version markers in `docs/index.html` |
| `python3 scripts/render_index_html.py [--check]` | Pre-render letter Markdown into `docs/index.html` |
| `python3 scripts/find_latest_ots.py <dir>` | Output info about the newest `.ots` proof |
| `bash scripts/verify-clearsign.sh` | Verify all `letter/*.asc` against `keys/FINGERPRINT` |
| `bash scripts/sign-and-export.sh <key> <file>` | Clear-sign a Markdown letter with GPG |

## Development conventions

### File naming
- Releases: `ASI-Letter-vYYYY.MM.DD.md` with matching `.asc` and `.asc.ots` — never rename these.
- Each `.asc` must have a corresponding `.asc.ots` before a release is considered complete.

### What is immutable
- `letter/*.asc` — signed releases; altering them breaks signature verification.
- `letter/*.asc.ots` — OTS proofs; only updated by OTS upgrade workflows, never by hand.
- `keys/FINGERPRINT` and `keys/alice-asi-publickey.asc` — only change on key rotation.

### What is generated (do not edit manually)
- `docs/letter.md` — regenerated by `sync_docs_with_latest.py`
- `docs/index.html` release markers (title, `data-release-version`, `<!-- release-version -->` comment, footer OTS status)
- `letter/RELEASES.json` — regenerated by `gen_releases_manifest.py`

### Fingerprint format
`FINGERPRINT` must be 40 uppercase hex characters, **no spaces**. The `sync-readme-fingerprint.yml` workflow auto-syncs the spaced display form in `README.md` from this file.

### Editor settings (`.editorconfig`)
- UTF-8 encoding, LF line endings, final newline, no trailing whitespace
- Markdown: 2-space indent

## CI / GitHub Actions workflows

All workflows share concurrency group `letter-artifacts-${{ github.ref }}` to prevent race conditions. Several jobs call `.github/scripts/wait_for_pages_idle.sh` before auto-committing to avoid interrupting GitHub Pages deployments.

| Workflow | Trigger | What it does |
|---|---|---|
| `verify-releases.yml` | push / PR | Installs GnuPG, runs `verify-clearsign.sh` — safety net for signature integrity |
| `releases-manifest.yml` | push to `letter/**` or `keys/**`, manual | Full release pipeline + auto-commit with `[manifest-auto]` tag |
| `auto-release-latest-letter.yml` | push to release paths | Creates/updates the GitHub Release tagged as "latest" |
| `ots-stamp-letter-asc.yml` | push to `letter/*.asc` | Stamps new `.ots` proofs and commits them |
| `ots-upgrade.yml` | scheduled / push / workflow_run | Upgrades existing OTS proofs and refreshes footer status |
| `ots-verify-upgrade.yml` | manual | Inspect + upgrade a single proof ad-hoc |
| `sync-readme-fingerprint.yml` | push to `keys/FINGERPRINT`, manual | Normalizes fingerprint and patches `README.md` |

### Auto-commit convention
Automated commits include tags in their messages to prevent re-triggering loops:
- `[manifest-auto]` — skips `releases-manifest.yml` re-run
- Check each workflow's `if:` conditions before modifying commit message formats.

## Safe vs. risky edits

### Safe
- Editing `README.md`, `CLAUDE.md`, `AGENTS.md` files
- Adding new `letter/ASI-Letter-vYYYY.MM.DD.md` files (unsigned source)
- Modifying `scripts/` logic (run `--check` after)
- Updating `docs/assets/`
- Editing CI workflow logic (keep concurrency groups and Pages waits intact)

### Requires care
- `docs/index.html` layout — avoid breaking automation markers (`<!-- release-version -->`, `data-release-version`, render markers)
- `.github/workflows/` — preserve concurrency groups, permissions, Pages wait calls, and auto-commit gating

### Do not modify without intentional release process
- Any `letter/*.asc` or `letter/*.asc.ots` file
- `keys/FINGERPRINT` or `keys/alice-asi-publickey.asc`
- `letter/RELEASES.json` (regenerate via scripts instead)

## Common failure modes to avoid

1. **Manual edits to generated files** — `docs/letter.md`, `docs/index.html` markers, `letter/RELEASES.json` drift from what scripts would produce.
2. **Fingerprint formatting** — adding spaces or lowercase to `keys/FINGERPRINT` breaks manifest generation and verification scripts.
3. **Missing `.ots` pairs** — every `.asc` must have a corresponding `.asc.ots` before publishing a GitHub Release.
4. **Breaking auto-commit loops** — changing commit message formats without updating workflow `if:` guards can cause infinite re-triggering.
5. **Removing Pages wait calls** — skipping `wait_for_pages_idle.sh` causes auto-commits to cancel in-flight Pages deployments.
6. **Modifying signed releases** — any change to a `.asc` file invalidates the OpenPGP signature.
